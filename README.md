# DrugSafe AI: Advanced Drug Interaction Platform

A professional, production-grade platform for analyzing drug interactions using **Machine Learning**, **Retrieval-Augmented Generation (RAG)**, and **GenAI** (via Groq/Llama).

## 🚀 Key Features

- **ML Severity Classifier**: Instant severity prediction with confidence scoring utilizing a **Scikit-learn RandomForest** model.
- **RAG Engine**: Contextual clinical grounding using ChromaDB and local embeddings to augment AI prompts with real-world clinical data.
- **AI Specialist Expert**: Lightning-fast, streaming expert analysis providing pharmacological explanations, generated by **Llama 3.1** via **Groq's LPU Interface**.
- **Real-time UI**: Professional React/Vite dashboard featuring streaming token animations and a premium dark-mode aesthetic.
- **Audit Trail**: Secure logging of every query and interaction analysis into a MySQL database for clinical review and auditing.
- **History Panel**: Persistent sidebar of previous queries for ease of use.

---

## 📂 Architecture & Tech Stack

- **Backend**: Python 3.11, FastAPI, SQLAlchemy, ChromaDB
- **Frontend**: React, Vite, Tailwind CSS / Custom CSS
- **Database**: MySQL 8.0, PyMySQL
- **AI & LLM**: Groq API (Llama 3.1 model specifically)
- **Machine Learning**: Scikit-Learn (RandomForestClassifier, HashingVectorizer)
- **Containerization**: Docker & Docker Compose

```text
drug-interaction-platform/
├── backend/               # FastAPI Backend Source
│   ├── api/               # API Routes & LLM/ML Clients (Groq integration)
│   ├── data/              # Internal ML Data & Vector Models
│   └── logs/              # API and System Logs
├── frontend/              # React App (Vite)
├── database/              # DB Initialization script (init.sql)
├── data/                  # External Data Sets for RAG Processing
├── docker-compose.yml     # Container Orchestration configs
└── .env                   # Environment Config
```

## 🛠️ Step-by-Step Setup (Local Development)

### 1. Database Setup
1. Open **MySQL Workbench** or your preferred SQL client.
2. Create a new database named `drug_interaction`.
3. The schema will be automatically created, or you can manually apply `database/init.sql`.

### 2. Environment Configuration
Create or update the `.env` file in the root of the project:
```env
# Groq API Configuration for Llama 3.1
GROQ_API_KEY=your_groq_api_key_here

# Database Configuration
MYSQL_USER=root
MYSQL_PASSWORD=your_mysql_password
MYSQL_DB=drug_interaction

# Note: For @ in password, use %40 in the DATABASE_URL below
# Local connection string example:
DATABASE_URL=mysql+pymysql://root:password_here@localhost:3306/drug_interaction
```

### 3. Backend Setup
The backend requires Python 3.9+. 
```powershell
# From the root directory
cd backend
pip install -r requirements.txt

# Start the FastAPI server
uvicorn api.main:app --reload
```
*Note: The backend will automatically index the CSV data into ChromaDB on startup if vector data is missing.*

### 4. Frontend Setup
Make sure you have Node.js 18+ installed.
```powershell
# Open a new terminal from the root directory
cd frontend
npm install
npm run dev -- --port 3001
```

## 🐳 Step-by-Step Setup (Docker Production)

For seamless deployment, the system is fully containerized.

1. Ensure **Docker Desktop** is running.
2. Verify your `.env` is populated with `GROQ_API_KEY`, `MYSQL_USER`, `MYSQL_PASSWORD`, and `MYSQL_DB`.
3. From the root directory, rebuild and start all containers:
```powershell
docker-compose up --build -d
```

### Docker Ports & Services
- **Frontend App**: [http://localhost:3000](http://localhost:3000)
- **Backend API Docs**: [http://localhost:8000/docs](http://localhost:8000/docs)
- **MySQL Database**: `localhost:3307` (Mapped from internal 3306 to avoid local conflicts)
